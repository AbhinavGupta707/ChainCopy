# ChainCopy — Native macOS Chain-Copy App PRD & Engineering Specification

**Document status:** Agent-ready v0.1  
**Date:** 2026-06-19  
**Target platform:** macOS, initially macOS 14+ recommended; macOS 13+ possible if dependencies allow  
**Working product names:** `ChainCopy`, `CopyStack`, `StackPaste`, `SnippetChain`  
**Primary objective:** Build a lightweight native macOS menu bar app that lets users collect multiple copied snippets, merge them into one clean paste-ready block, paste once, and optionally reset automatically.

---

## 1. Executive summary

This product should be a **small native macOS menu bar utility**, not a full clipboard manager. The primary workflow is:

```text
Copy first thing normally
Append more things using a shortcut, append mode, or current-clipboard append
Paste one combined block with a configurable separator
Reset automatically
```

The product should be optimized for developers, AI-heavy users, researchers, and writers who frequently copy several blocks from ChatGPT, Claude, Codex, Cursor, GitHub, docs, terminal output, Slack, Notion, PDFs, and web apps.

The core differentiator is not “clipboard history.” It is:

```text
Many copied snippets in → one clean composed paste out → reset
```

Existing products prove demand and implementation feasibility:

- **Batch Clipboard** is a free/open-source MIT macOS menu bar utility built in Swift. It queues multiple clipboard items using `Control + Command + C` and replays them using `Control + Command + V`.
- **Hammerspoon** is a free/open-source MIT macOS automation app that exposes keyboard, pasteboard, event tap, and hotkey APIs through Lua. It is a strong prototyping reference.
- **VibeMeter / CodexBar** are useful references for minimal native menu bar app UX, Swift/SwiftUI/AppKit architecture, release automation, Sparkle updates, and agent-friendly code organization.
- **Maccy, SaneClip, PasteBar, Raycast extensions, Alfred, Keyboard Maestro** validate the broader clipboard utility market, but most are either generic history managers or automation platforms rather than a focused chain-copy composer.

Recommended product direction:

1. **Do not start by intercepting every normal `Cmd+C`.** That increases privacy, permission, and reliability risk.
2. **Do use explicit user actions**: append selected text, append current clipboard, append mode, paste and reset.
3. **Support copy buttons through Append Mode** and “Append Current Clipboard,” because web copy buttons write to the system clipboard rather than exposing a selected-text event.
4. **Ship direct-download first** with Developer ID notarization and Sparkle updates. Consider a Mac App Store edition later with degraded auto-paste behavior if sandbox rules become a blocker.
5. **Use Batch Clipboard and Hammerspoon as references**, but build a clean Swift codebase tailored to the join/compose/paste-reset workflow.

## 2. Product thesis

Users often collect multiple fragments before pasting somewhere else:

- code blocks from ChatGPT / Claude / Codex;
- terminal logs and stack traces;
- docs snippets;
- issue tracker details;
- Slack / email quotes;
- research passages;
- links and notes;
- repeated copy buttons in AI tools.

Today they usually do one of these:

1. switch back and forth repeatedly: copy → paste → copy → paste;
2. use a temporary scratchpad;
3. use a generic clipboard history app;
4. use Alfred/Keyboard Maestro/Raycast if already in that ecosystem;
5. accept a queue-style tool that pastes one item at a time.

The opportunity is a focused utility that makes the common “collect snippets and paste one final block” workflow fast, visible, and private.

## 3. Research summary

### 3.1 Batch Clipboard

**Repository:** `https://github.com/jpmhouston/Batch-Clipboard`  
**Docs:** `https://batchclipboard.bananameter.lol/`  
**License:** MIT  
**Language:** Swift-heavy native macOS app  
**Distribution:** GitHub releases, Mac App Store, Homebrew tap/cask references

What Batch Clipboard already does well:

- Provides a menu bar utility for copying a batch of multiple items and pasting them in order.
- Default workflow is `Control + Command + C` to copy into the batch and `Control + Command + V` to paste from the batch.
- Once in batch mode, any change to the regular system clipboard can be seen and added to the active batch. Their docs explicitly mention frontmost app Copy/Cut, right-click Copy/Cut, and other app features that write to the regular system clipboard.
- It avoids always-on monitoring by default: when history features are off, it only monitors the clipboard while batch mode is active.
- It ignores confidential/temporary pasteboard types by default: `org.nspasteboard.TransientType`, `org.nspasteboard.ConcealedType`, `org.nspasteboard.AutoGeneratedType`, plus additional known custom types.
- It has menu bar count UX, batch mode state, and settings references that are highly relevant.

What it does not appear to do as the primary workflow:

- Join all snippets into one plain-text block with a configurable separator.
- Paste the joined block once and reset.
- Focus specifically on AI/code/research snippet composition.
- Provide a “compose preview/edit/reorder” UX optimized for one-block output.

Use as reference for:

- menu bar app structure;
- clipboard monitoring mode;
- pasteboard type exclusion settings;
- batch state and menu count behavior;
- Sparkle / Mac App Store dual distribution ideas;
- testing and build setup.

Do not blindly fork as product unless the team wants the fastest possible prototype. A clean native app may better serve the differentiated product.

### 3.2 Hammerspoon

**Repository:** `https://github.com/Hammerspoon/hammerspoon`  
**Docs:** `https://www.hammerspoon.org/docs/`  
**License:** MIT

Hammerspoon is a macOS automation app that bridges macOS system APIs into Lua scripts. It can create global hotkeys, send keystrokes, inspect/manipulate pasteboards, and watch pasteboard changes.

Relevant APIs and concepts:

- `hs.hotkey`: create/manage global keyboard shortcuts.
- `hs.eventtap`: observe/send keyboard/mouse events; exposes secure input detection.
- `hs.pasteboard`: inspect/manipulate pasteboards, read/write contents, get `changeCount`.
- `hs.pasteboard.callbackWhenChanged`: capture results after issuing a programmatic copy.
- `hs.pasteboard.watcher`: watch pasteboard changes via polling because macOS does not offer pasteboard change notifications.

Use as reference for:

- a 1–2 day prototype before building native Swift;
- event timing patterns;
- `Cmd+C` simulation followed by pasteboard-change waiting;
- append mode behavior;
- debugging race conditions.

Do not ship Hammerspoon as the product. It is too user-config/config-code oriented for a consumer/prosumer utility.

### 3.3 VibeMeter / CodexBar by Peter Steinberger

**VibeMeter repository:** `https://github.com/steipete/VibeMeter`  
**VibeMeter website:** `https://vibemeter.ai/`  
**CodexBar repository:** `https://github.com/steipete/codexbar`  
**CodexBar website:** `https://codexbar.app/`  
**License:** MIT for referenced repos, based on repository pages

Relevant lessons:

- Small native menu bar apps can be built with Swift/SwiftUI/AppKit and a mostly invisible, efficient UX.
- SwiftUI handles much of the UI, but AppKit may still be necessary for menu bar popovers, panels, status item behavior, and settings quirks.
- Agent-friendly development benefits from small files, protocol boundaries, testability, and release scripts.
- Production macOS apps need release build testing, signing, notarization, Sparkle update feed correctness, and careful memory/window lifecycle handling.
- A menu bar app should use very low CPU while idle.

Use as reference for:

- Swift 6 architecture;
- `@MainActor` UI isolation;
- actor-based background services;
- test coverage culture;
- release automation scripts;
- Sparkle update integration;
- no-Dock-icon menu bar utility structure.

### 3.4 Maccy, SaneClip, PasteBar, Raycast, Alfred, Keyboard Maestro

Useful competitive/reference observations:

- **Maccy** is a lightweight, native, MIT/open-source clipboard history manager. It validates user appetite for minimal clipboard tooling and local-first privacy.
- **Batch Clipboard** was originally a Maccy fork and later refactored away from upstream structure.
- **SaneClip** is useful as a privacy-first and commercial direct-download/App Store comparison. It notes a direct-download version can support auto-paste while the Mac App Store version may need to fall back to “copy to clipboard and ask user to press `Cmd+V`” due to sandbox restrictions.
- **Raycast Append Clipboard** validates append/prepend selected text with configurable separators as a narrow extension workflow.
- **Alfred Powerpack** validates `Cmd+C+C` style clipboard merging as a loved feature, but it is paid and bundled into a broader launcher/productivity suite.
- **Keyboard Maestro** validates custom macro demand, but is too broad/complex for users who just want this one feature.

## 4. Product positioning

### 4.1 One-line positioning

**ChainCopy is a tiny, private macOS menu bar app that turns many copied snippets into one clean paste-ready block.**

### 4.2 Differentiation

Not:

```text
Another clipboard history manager.
```

Instead:

```text
A chain-copy composer for AI, code, research, and writing workflows.
```

### 4.3 Target users

Primary:

- developers;
- AI power users;
- prompt engineers;
- researchers;
- technical writers;
- product/support people collecting evidence from multiple tools.

Secondary:

- students;
- analysts;
- lawyers/paralegals;
- consultants;
- anyone assembling multi-part notes.

### 4.4 Core jobs-to-be-done

1. **When I copy multiple code blocks from AI tools, I want to paste them as one clean block so I do not bounce between apps.**
2. **When I collect logs/errors/docs, I want one shortcut to append each snippet and one shortcut to paste everything.**
3. **When I click AI/web “Copy” buttons, I want those copied blocks to be captured without selecting text manually.**
4. **When I paste the final block, I want the app to reset so the next chain starts cleanly.**
5. **When I use a clipboard tool, I want to trust that it is local-only and not silently storing passwords.**

## 5. Product principles

1. **Explicit over creepy.** Clipboard capture should happen because the user pressed an app hotkey or enabled Append Mode.
2. **Local-only by default.** No cloud sync, no content telemetry, no remote clipboard storage.
3. **Small and visible.** Menu bar count/state should always reveal whether the app is collecting.
4. **Fast keyboard UX first.** Mouse UI is for preview/settings, not required for normal use.
5. **One clean output.** The default output is one plain-text block joined by a blank line.
6. **Reset safely.** Internal chain resets after paste; system clipboard should usually retain the composed block to avoid paste timing issues.
7. **Composable workflows.** Let users choose between selected-text append, append current clipboard, append mode, and advanced modes.
8. **No generic clipboard-history creep for MVP.** Avoid competing with Maccy/Paste/PasteBar.

## 6. MVP workflow specification

### 6.1 Default shortcuts

All shortcuts must be user-customizable.

| Action | Default shortcut | Description |
|---|---:|---|
| Append selected text | `Ctrl + Cmd + C` | Simulates `Cmd+C`, waits for clipboard change, appends copied selected text. If chain is empty and clipboard already had a previous text item, seed chain with that previous clipboard text first. |
| Append current clipboard | `Ctrl + Cmd + Shift + C` | Appends whatever text is currently on the system clipboard. Useful after clicking web/app Copy buttons. |
| Toggle Append Mode | `Ctrl + Cmd + A` | While active, every qualifying system clipboard text change is appended automatically. Useful for repeated AI copy buttons. |
| Paste chain and reset | `Ctrl + Cmd + V` | Joins snippets, writes joined block to clipboard, sends paste, clears internal chain. |
| Clear chain | `Ctrl + Cmd + X` | Clears internal chain without pasting. |
| Show ChainCopy popover | configurable | Opens menu bar popover/preview. Optional for MVP. |

Avoid using bare `Ctrl+V` as the default. It can conflict with terminals and app-specific keybindings. `Ctrl+Cmd+V` is safer and symmetrical with `Ctrl+Cmd+C`.

### 6.2 Workflow A — selected text, no copy buttons

Desired user flow:

```text
1. Select Alpha
2. Cmd+C
3. Select Beta
4. Ctrl+Cmd+C
5. Select Gamma
6. Ctrl+Cmd+C
7. Ctrl+Cmd+V
```

Expected paste:

```text
Alpha

Beta

Gamma
```

Implementation behavior:

- On first normal `Cmd+C`, ChainCopy does not need to intercept anything. Alpha is simply on the system clipboard.
- On first `Ctrl+Cmd+C`, app reads `previousClipboard = Alpha`, then simulates `Cmd+C` to copy Beta, waits for pasteboard change, and appends `[Alpha, Beta]` if the internal chain is empty.
- On second `Ctrl+Cmd+C`, app appends Gamma.
- On `Ctrl+Cmd+V`, app composes and pastes.

This satisfies the user’s desire that the first copy can just be normal `Cmd+C`.

### 6.3 Workflow B — selected text with explicit start

Alternative for users who want clearer state:

```text
1. Select Alpha
2. Ctrl+Cmd+C
3. Select Beta
4. Ctrl+Cmd+C
5. Ctrl+Cmd+V
```

This can work because `Ctrl+Cmd+C` copies selected text and appends it, starting the chain if empty.

### 6.4 Workflow C — web/app Copy buttons

Copy buttons do not expose selected text. They write directly to the macOS system clipboard. Therefore the app should support two flows.

#### Flow C1 — Append Current Clipboard

```text
1. Click Copy button in ChatGPT/Claude/Codex
2. Ctrl+Cmd+Shift+C
3. Click another Copy button
4. Ctrl+Cmd+Shift+C
5. Ctrl+Cmd+V
```

This is reliable because ChainCopy reads the current clipboard after the web/app button has written to it.

#### Flow C2 — Append Mode

```text
1. Ctrl+Cmd+A to turn Append Mode on
2. Click Copy button in ChatGPT
3. Click Copy button in Claude
4. Cmd+C selected text from docs
5. Click Copy button in GitHub
6. Ctrl+Cmd+A to turn Append Mode off
7. Ctrl+Cmd+V to paste final block and reset
```

While Append Mode is active, ChainCopy watches the system pasteboard and appends qualifying text changes.

This is the best UX for repeated AI/web copy buttons.

### 6.5 Workflow D — paste and reset

Default paste behavior:

1. Compose `chain.items.map(\.text).joined(separator: settings.separator)`.
2. Write composed output to `NSPasteboard.general`.
3. Send normal `Cmd+V` to the frontmost app.
4. Wait a safe delay, e.g. 500–1000 ms.
5. Clear internal chain.
6. Leave the composed output on the system clipboard by default.

Why leave the composed output on the clipboard?

- Some apps paste asynchronously.
- Clearing/restoring too fast can break paste.
- Users often expect `Cmd+V` after that to paste the same final block again.

Advanced setting:

```text
After paste: [ Leave composed block on clipboard | Restore previous clipboard | Clear clipboard ]
```

Default should be **Leave composed block on clipboard**.

## 7. Non-MVP / future workflows

### 7.1 Alfred-style `Cmd+C+C`

Optional future feature:

```text
Cmd+C once = normal copy
Cmd+C twice quickly = append copied selection to previous clipboard/chain
```

This requires lower-level event monitoring and careful handling of timing, repeats, secure input, keyboard layouts, and app conflicts. It is likely a Pro/advanced feature after MVP.

### 7.2 Modifier-click Copy button

The original idea of holding Control while clicking a Copy button is not recommended for MVP because Control-click is a standard macOS secondary-click/context-menu gesture.

Possible future experiment:

```text
Hold Option while clicking a Copy button → next clipboard change appends
```

Implementation would require mouse/key event monitoring and a “next clipboard change is intentional” window. This is more fragile than Append Mode.

### 7.3 Rich text / images / files

MVP should focus on plain text. Later versions may support:

- rich text preservation;
- Markdown wrapping;
- images;
- file references;
- OCR.

## 8. Functional requirements

### 8.1 P0 — MVP required

#### Chain engine

- Maintain ordered list of text snippets.
- Join snippets with a configurable separator.
- Default separator: one blank line, `\n\n`.
- Store optional metadata: source app, capture method, timestamp, content hash.
- Do not persist clipboard contents to disk by default.
- Duplicate suppression: ignore if new text hash equals last snippet hash, unless user disables.
- Max snippet length setting, default maybe 1 MB per item.
- Max chain items setting, default maybe 100.

#### Hotkeys

- User-customizable global hotkeys.
- Provide default shortcuts listed above.
- Warn on conflicts.
- Allow disabling any hotkey.

#### Append selected text

- Save current clipboard string and change count.
- Send `Cmd+C` to frontmost app.
- Wait for pasteboard `changeCount` to change, with timeout.
- Read new string.
- If chain empty and prior clipboard is non-empty and not equal to new string, seed chain with prior clipboard first.
- Append new string.
- Show unobtrusive notification or menu bar count update.
- If copy times out, show concise failure and do not mutate chain.

#### Append current clipboard

- Read current clipboard string.
- Append if non-empty, not excluded, and not duplicate.
- Start chain if empty.

#### Append Mode

- User toggles active/inactive explicitly.
- Poll pasteboard only while Append Mode is active.
- Every qualifying new text clipboard change appends to chain.
- Ignore app’s own pasteboard writes.
- Ignore confidential/temporary pasteboard types.
- Show visible menu bar active state.

#### Paste chain and reset

- Compose joined output.
- Write composed output to system pasteboard.
- Send `Cmd+V` or provide fallback to copy-only mode if auto-paste permission is unavailable.
- Clear internal chain after paste.
- Leave composed output on system clipboard by default.

#### Menu bar UI

- Menu bar icon always visible while app runs.
- Shows idle, collecting count, append mode, paused, and error states.
- Popover/menu with current count, preview snippets, append mode toggle, paste/reset, clear, settings, quit.

#### Privacy controls

- Local-only.
- No clipboard-content analytics.
- No persistent clipboard history by default.
- Ignore known sensitive pasteboard types by default.
- User-configurable ignored apps and pasteboard types.
- Manual pause and clear.

### 8.2 P1 — sellable utility

- Preview/edit/reorder/delete individual snippets.
- Separator presets.
- Per-profile workflows.
- Source labels such as `[ChatGPT]`, `[Terminal]`, `[GitHub]`.
- Markdown code-fence mode.
- App-specific rules.
- Export chain to file.
- Debug panel.
- Launch at login.
- Signed direct-download updates.
- In-app license/upgrade flow.

### 8.3 P2 — advanced/future

- `Cmd+C+C` merge mode.
- Modifier-click capture mode.
- Browser extension for deeper copy-button integration.
- Rich text/images/files.
- OCR capture.
- Saved chains/templates.
- Optional iCloud sync.
- AI cleanup/summarization.

## 9. Non-functional requirements

### Performance

- Idle CPU near zero.
- No polling unless Append Mode or optional history/watch mode is active.
- Append Mode polling interval default: 250 ms; user-adjustable advanced setting.
- Memory target: <75 MB for MVP.
- Handle large snippets gracefully.

### Reliability

- Never destroy user clipboard unexpectedly.
- Do not clear system clipboard immediately after paste by default.
- Avoid duplicate appends from own writes.
- Handle copy timeouts.
- Handle non-text clipboard content.
- Handle secure input or permission failure with clear messaging.

### Privacy/security

- No clipboard contents in logs by default.
- Debug logs store lengths, hashes, source app, and event types; full content logging requires explicit temporary opt-in.
- Ignore sensitive pasteboard types.
- Ignore common password/security apps by default.
- No network calls required for app functionality.
- If telemetry is ever added, it must be opt-in and must never include clipboard content.

### Accessibility and UX

- Keyboard-first.
- VoiceOver labels for controls.
- High-contrast menu bar states.
- Non-intrusive notifications.
- Clear onboarding for permissions and shortcuts.

## 10. State machine

### States

```text
Idle
  No internal chain, Append Mode off.

Collecting
  Internal chain has >= 1 items, Append Mode off.

AppendModeActive
  Pasteboard watcher active; app appends qualifying clipboard changes.

Pasting
  App is writing composed output and sending paste; own writes ignored.

Paused
  App ignores clipboard changes and hotkey capture except resume/clear/settings.

Error
  Permission denied, pasteboard failure, hotkey conflict, or secure input issue.
```

### Events

```text
normalClipboardChanged
appendSelectedHotkey
appendCurrentClipboardHotkey
toggleAppendModeHotkey
pasteAndResetHotkey
clearHotkey
permissionGranted
permissionDenied
ownPasteboardWrite
copyTimeout
excludedPasteboardType
excludedSourceApp
```

### Key transitions

```text
Idle + appendCurrentClipboard → Collecting
Idle + appendSelectedHotkey → Collecting
Idle + toggleAppendMode → AppendModeActive
Collecting + appendSelectedHotkey → Collecting
Collecting + appendCurrentClipboard → Collecting
Collecting + pasteAndReset → Pasting → Idle
AppendModeActive + normalClipboardChanged → AppendModeActive with item appended
AppendModeActive + pasteAndReset → Pasting → Idle
Any + clearHotkey → Idle
Any + pause → Paused
Paused + resume → previous safe state or Idle
```

## 12. Technical architecture

### 12.1 Recommended stack

```text
Language: Swift 6
UI: SwiftUI + AppKit where necessary
App type: macOS menu bar utility, no Dock icon by default
Clipboard: NSPasteboard
Hotkeys: KeyboardShortcuts package
Launch at login: LaunchAtLogin-Modern package
Updates for direct distribution: Sparkle 2
Settings: SwiftUI Settings scene, optional Sindre Sorhus Settings package
Logging: OSLog
Tests: XCTest or Swift Testing + protocol-based mocks
Distribution: Developer ID signed + notarized DMG/ZIP; GitHub Releases; Sparkle appcast
```

### 12.2 Core modules

```text
AppShell
  Owns app lifecycle, menu bar status item, popover, settings, onboarding.

ChainEngine
  Pure Swift domain model for clips, ordering, composing, clearing, duplicate suppression.

ClipboardService
  NSPasteboard wrapper for reading/writing strings, reading content types, observing changeCount.

CaptureService
  Implements append selected text, append current clipboard, seed-from-previous-clipboard behavior.

AppendModeMonitor
  Polls pasteboard changes only when Append Mode is active.

PasteService
  Writes composed block, triggers paste, resets chain safely.

HotkeyManager
  Registers and routes global shortcuts.

PermissionManager
  Checks/request Accessibility/Input Monitoring/Automation as needed.

SourceAppService
  Reads frontmost app bundle ID/name, applies ignored-app rules.

SettingsStore
  Stores preferences using UserDefaults or a typed Defaults wrapper.

DebugLogger
  Local diagnostics without clipboard content by default.

LicensingService (P1)
  Handles direct license or App Store purchase state.
```

### 12.6 Clipboard service interface


Implementation notes:

### 12.7 Append selected text algorithm

```text
appendSelectedText():
  ensure capture permissions if needed
  let previousSnapshot = clipboard.snapshot()
  let previousText = clipboard.readString()
  let oldChangeCount = clipboard.changeCount
  let sourceAppBefore = sourceAppService.frontmostApplication()

  eventSynthesizer.sendCommandC(to: sourceAppBefore)

  changed = await clipboard.waitForChange(after: oldChangeCount, timeout: 1.0)
  if !changed:
      notify copy timeout
      return

  let newTypes = clipboard.readTypes()
  if sensitiveFilter.shouldIgnore(types: newTypes, sourceApp: sourceAppBefore):
      notify ignored sensitive content
      return

  guard let newText = clipboard.readString(), !newText.isEmpty else:
      notify unsupported or empty clipboard
      return

  if chain.isEmpty && settings.seedFromPreviousClipboardOnFirstAppend:
      if previousText exists && previousText != newText && !sensitive(previousSnapshot):
          chain.append(previousText, metadata: .seedFromExistingClipboard)

  chain.append(newText, metadata: .appendSelectedTextHotkey)
  update menu bar count
```

Important edge cases:

- If the selected text is the same as the previous clipboard, do not seed duplicate.
- If `Cmd+C` did not change the clipboard, show timeout or “copy unchanged” message.
- If secure input is active, copying may fail; surface this.
- If frontmost app blocks synthetic copy, offer fallback: “Press `Cmd+C`, then Append Current Clipboard.”

### 12.8 Append current clipboard algorithm

```text
appendCurrentClipboard():
  let types = clipboard.readTypes()
  let sourceApp = sourceAppService.frontmostApplication()

  if sensitiveFilter.shouldIgnore(types, sourceApp): return
  guard let text = clipboard.readString(), !text.isEmpty else: return

  chain.append(text, metadata: .appendCurrentClipboardHotkey)
  update menu bar count
```

This is the safest and simplest mechanism for copy buttons.

### 12.9 Append Mode monitor algorithm

```text
startAppendMode():
  appendMode = true
  lastSeenChangeCount = clipboard.changeCount
  start timer interval 0.25s
  update menu bar state

onTimerTick():
  guard appendMode else return
  let currentChangeCount = clipboard.changeCount
  guard currentChangeCount != lastSeenChangeCount else return
  lastSeenChangeCount = currentChangeCount

  if ownWriteTracker.contains(currentChangeCount): return
  let sourceApp = sourceAppService.frontmostApplication()
  let types = clipboard.readTypes()
  if sensitiveFilter.shouldIgnore(types, sourceApp): return

  guard let text = clipboard.readString(), !text.isEmpty else return
  chain.append(text, metadata: .appendModeClipboardChange)
```

Because macOS does not expose native pasteboard-change notifications, polling is expected. Hammerspoon and many clipboard managers use polling-like patterns.

### 12.10 Paste chain and reset algorithm

```text
pasteChainAndReset():
  guard !chain.isEmpty else:
      notify no snippets
      return

  let output = chain.compose()
  let previousSnapshot = clipboard.snapshot()

  ownWriteTracker.begin()
  clipboard.writeString(output)
  ownWriteTracker.record(clipboard.changeCount)

  if permissions.canAutoPaste:
      eventSynthesizer.sendCommandV()
      wait 0.5-1.0s
      chain.clear()
      if settings.restorePreviousClipboardAfterPaste:
          clipboard.restore(previousSnapshot)
      else if settings.clearClipboardAfterPaste:
          clipboard.writeString("")
      else:
          leave output on clipboard
  else:
      notify "Joined block copied. Press Cmd+V to paste."
      chain.clear() only if user setting says clear after copy-only
```

Implementation:

```swift
struct SensitivePasteboardFilter {
    let ignoredTypes: Set<String>
    let ignoredBundleIDs: Set<String>
    let ignoredAppNameSubstrings: [String]

    func shouldIgnore(types: [String], sourceApp: SourceApp?) -> Bool {
        if !Set(types).isDisjoint(with: ignoredTypes) { return true }
        if let bundleID = sourceApp?.bundleID, ignoredBundleIDs.contains(bundleID) { return true }
        if let name = sourceApp?.name?.lowercased() {
            if ignoredAppNameSubstrings.contains(where: { name.contains($0.lowercased()) }) {
                return true
            }
        }
        return false
    }
}
```

Caveat: browser extensions often write via the browser process, so app-name ignore alone is imperfect. Pasteboard type exclusions and explicit Append Mode reduce risk but cannot perfectly identify every password copy.

## 13. Permissions and macOS platform constraints

### 13.1 Hotkeys

Use `KeyboardShortcuts` for global hotkeys. It is designed for user-customizable global keyboard shortcuts, stores shortcuts, warns on conflicts, and is sandbox/Mac App Store compatible. It should not require permission dialogs for basic registered shortcuts.

### 13.2 Copy-selected and paste automation

To implement `appendSelectedText`, the app needs to send a `Cmd+C` event to the frontmost app. To implement `pasteAndReset`, it needs to send `Cmd+V`. Depending on implementation and distribution path, this may require Accessibility or be restricted under sandbox/App Store rules.

Design fallback:

- If auto-copy/paste permission unavailable:
  - `Append Selected Text` can fall back to instruction: “Press `Cmd+C`, then Append Current Clipboard.”
  - `Paste Chain & Reset` can write joined output to clipboard and show: “Joined block copied. Press `Cmd+V`.”

### 13.3 Input Monitoring / CGEventTap

Do not require arbitrary key-event monitoring for MVP. Only consider `CGEventTap` for future advanced modes such as `Cmd+C+C` detection or modifier-click append. If implemented, use explicit onboarding and permissions. Apple developer guidance suggests `CGEventTap` with Input Monitoring is the path for sandbox-compatible keyboard listening, but it is lower-level and more complex than registered hotkeys.

### 13.4 Secure Input

When secure input is active, event taps or synthetic event workflows may fail. The app should detect likely failure via timeout and show a clear non-scary message:

```text
Could not copy from the current app. Try Cmd+C, then Append Current Clipboard.
```

### 13.5 Pasteboard privacy

macOS clipboard access is privacy-sensitive. Design around explicit user actions:

- read clipboard in response to app hotkey;
- read clipboard while Append Mode is explicitly visible/on;
- avoid silent always-on clipboard scraping;
- never persist contents unless user opts in.

## 19. Known risks and mitigations

### Risk: copy buttons cannot be semantically detected

A web Copy button is just a clipboard write. The app cannot know “this came from a Copy button” unless a browser extension or accessibility inspection is added.

Mitigation:

- Use Append Mode.
- Use Append Current Clipboard.
- Avoid modifier-click as MVP.

### Risk: global event monitoring triggers privacy concerns

Mitigation:

- Do not globally monitor all keys in MVP.
- Use registered hotkeys.
- Make Append Mode explicit and visible.

### Risk: synthetic `Cmd+C` / `Cmd+V` is unreliable or permission-gated

Mitigation:

- Wait on pasteboard `changeCount`, not fixed delay only.
- Provide fallback to append current clipboard and copy-only paste.
- Clear error messages.

### Risk: Mac App Store sandbox blocks auto-paste

Mitigation:

- Direct-download first.
- MAS edition later with fallback behavior.

### Risk: generic clipboard market is crowded

Mitigation:

- Avoid generic history positioning.
- Focus on AI/code/research composition.
- Make UX faster and simpler than history managers.

### Risk: accidentally capturing sensitive data

Mitigation:

- No persistent history.
- Sensitive pasteboard type filtering.
- Ignored apps.
- Pause mode.
- Clear chain on lock/sleep.
- No content logs by default.

## 22. Example minimal Hammerspoon prototype behavior

This is not production code, but it shows the target behavior to validate before native implementation.

```lua
local pasteboard = hs.pasteboard
local eventtap = hs.eventtap
local hotkey = hs.hotkey
local timer = hs.timer

local chain = {}
local separator = "\n\n"
local appendMode = false
local lastChangeCount = pasteboard.changeCount()
local ownWrite = false

local function appendText(text)
  if not text or text == "" then return end
  if #chain > 0 and chain[#chain] == text then return end
  table.insert(chain, text)
  hs.alert.show("ChainCopy: " .. tostring(#chain) .. " snippets")
end

local function joined()
  return table.concat(chain, separator)
end

hotkey.bind({"ctrl", "cmd", "shift"}, "C", function()
  appendText(pasteboard.getContents())
end)

hotkey.bind({"ctrl", "cmd"}, "C", function()
  local previous = pasteboard.getContents()
  local oldCount = pasteboard.changeCount()
  eventtap.keyStroke({"cmd"}, "c", 0)
  hs.timer.doAfter(0.1, function()
    hs.pasteboard.callbackWhenChanged(1.0, function(changed)
      if changed then
        local newText = pasteboard.getContents()
        if #chain == 0 and previous and previous ~= "" and previous ~= newText then
          appendText(previous)
        end
        appendText(newText)
      else
        hs.alert.show("Copy timeout")
      end
    end)
  end)
end)

hotkey.bind({"ctrl", "cmd"}, "A", function()
  appendMode = not appendMode
  lastChangeCount = pasteboard.changeCount()
  hs.alert.show("Append Mode " .. (appendMode and "On" or "Off"))
end)

hs.timer.doEvery(0.25, function()
  if not appendMode or ownWrite then return end
  local count = pasteboard.changeCount()
  if count ~= lastChangeCount then
    lastChangeCount = count
    appendText(pasteboard.getContents())
  end
end)

hotkey.bind({"ctrl", "cmd"}, "V", function()
  if #chain == 0 then return end
  ownWrite = true
  pasteboard.setContents(joined())
  eventtap.keyStroke({"cmd"}, "v", 0)
  timer.doAfter(0.75, function()
    chain = {}
    ownWrite = false
    hs.alert.show("Chain pasted and reset")
  end)
end)

hotkey.bind({"ctrl", "cmd"}, "X", function()
  chain = {}
  hs.alert.show("Chain cleared")
end)
```

## 24. Source evidence notes

Important facts used in this PRD:

- Batch Clipboard’s docs describe its basic workflow as `Control + Command + C` to copy multiple items and `Control + Command + V` to paste them in order.
- Batch Clipboard’s docs state that once already in batch mode, any change to the system clipboard is seen, including right-click Copy/Cut or other app features that add to the system clipboard.
- Batch Clipboard’s docs state it only monitors clipboard while batch mode is active when history features are off.
- Batch Clipboard’s docs list ignored confidential/temporary pasteboard types.
- Hammerspoon describes itself as a bridge between macOS and Lua and exposes pasteboard, hotkey, and event APIs.
- Hammerspoon pasteboard watcher docs state macOS does not provide pasteboard notifications, so it polls.
- Hammerspoon pasteboard docs expose `changeCount` and `callbackWhenChanged` patterns.
- Apple docs and forums indicate global key event monitoring/access can involve Accessibility or Input Monitoring depending on API choice.
- KeyboardShortcuts advertises customizable global keyboard shortcuts with sandbox/Mac App Store compatibility and no permission dialogs.
- Sparkle docs recommend HTTPS, code signing/notarization, and EdDSA signatures.
- VibeMeter/CodexBar provide relevant native menu bar app references for SwiftUI/AppKit, release automation, and agent-friendly architecture.

## 25. Final recommendation

Build **ChainCopy** as a clean native Swift menu bar app.

Use this MVP UX:

```text
Cmd+C
  Normal first copy; no app interception needed.

Ctrl+Cmd+C
  Append selected text; seeds chain from existing clipboard if needed.

Ctrl+Cmd+Shift+C
  Append current clipboard; works after web/app Copy buttons.

Ctrl+Cmd+A
  Toggle Append Mode; best for repeated ChatGPT/Claude/Codex copy buttons.

Ctrl+Cmd+V
  Paste joined block with blank-line separators and reset internal chain.
```

Do not build a generic history manager. Do not make always-on clipboard scraping the default. Do not rely on Control-clicking web Copy buttons. Focus the app on one workflow and make that workflow feel fast, private, and reliable.
