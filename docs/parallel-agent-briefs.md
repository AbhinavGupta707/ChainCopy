# Parallel Agent Briefs

Date: 2026-06-19

Use these as starting prompts for parallel implementation sessions after the repository is pushed. Each agent should branch from `main`, keep changes scoped, run `xcodegen generate`, and verify with:

```sh
xcodebuild -project ChainCopy.xcodeproj -scheme ChainCopy -destination 'platform=macOS' -derivedDataPath build/DerivedData test
```

## Agent A: Clipboard Capture Engine

Branch: `feature/capture-engine`

Goal: Turn the current plain-text polling scaffold into a robust capture layer.

Scope:
- Add pasteboard type modeling for plain text, rich text, URLs, files, and unsupported entries.
- Add ignored pasteboard marker handling for transient, concealed, autogenerated, and app-owned writes.
- Add source-app exclusion rules and max-size safeguards.
- Add focused unit tests around dedupe, filtering, and self-write suppression.

Avoid:
- UI redesign.
- Global hotkeys.
- Persistence beyond in-memory models unless absolutely required for tests.

## Agent B: Hotkeys And Paste Automation

Branch: `feature/hotkeys-paste`

Goal: Add configurable keyboard control for capture, compose, paste, pause, and quick UI.

Scope:
- Add `KeyboardShortcuts` or equivalent after confirming package compatibility.
- Add default shortcuts and Settings UI for reassignment.
- Add Accessibility permission detection and onboarding state.
- Add synthetic paste flow behind a permission gate.
- Add diagnostics when paste automation is unavailable.

Avoid:
- Changing pasteboard storage models unless Agent A has landed.
- Shipping hidden shortcuts without Settings visibility.

## Agent C: Premium UI And Interaction

Branch: `feature/premium-ui`

Goal: Build the sellable menu bar and composer experience.

Scope:
- Replace the scaffold UI with a polished menu bar window and composer panel.
- Add empty, paused, active, error, and permission-required states.
- Add reorder, remove, pin, search/filter, and separator controls.
- Keep native macOS visual language: materials, compact density, keyboard focus, contextual menus.
- Add accessibility labels and keyboard navigation.

Avoid:
- Decorative marketing screens.
- Overly custom chrome that fights native macOS behavior.

## Agent D: Persistence And Privacy

Branch: `feature/persistence-privacy`

Goal: Make clipboard state durable, predictable, and private.

Scope:
- Decide and implement the local store format.
- Add retention settings, clear-on-quit option, and sensitive-content filters.
- Add migration-safe models and tests.
- Add privacy settings for ignored apps, ignored pasteboard types, and excluded patterns.

Avoid:
- Cloud sync.
- Analytics that includes clipboard content.

## Agent E: Distribution And Release

Branch: `feature/distribution`

Goal: Prepare direct-download release infrastructure.

Scope:
- Add Sparkle dependency and update channel scaffolding.
- Add release build configuration, signing docs, notarization scripts, and artifact packaging.
- Add GitHub Actions workflow for build/test and a separate release workflow stub.
- Document required Apple Developer credentials and secrets.

Avoid:
- Storing certificates or secrets in the repo.
- Finalizing licensing provider integration before provider choice.

## Agent F: Commercial And Licensing Evaluation

Branch: `feature/licensing-eval`

Goal: Pick the paid-app infrastructure path before we wire activation UI.

Scope:
- Compare Lemon Squeezy, Polar, RevenueCat for macOS, Paddle, and Gumroad-style license flows.
- Recommend one provider for direct download.
- Define trial, activation, offline grace, refund/support, and machine-limit policies.
- Produce an implementation ticket list and required secrets.

Avoid:
- Implementing a provider SDK before decision.
- Blocking free/internal beta builds on licensing.

## Agent G: QA, Telemetry, And Diagnostics

Branch: `feature/qa-diagnostics`

Goal: Add production-grade observability without leaking user data.

Scope:
- Add structured `Logger` categories for capture, compose, permissions, hotkeys, and release checks.
- Add an in-app diagnostics export that excludes clipboard contents by default.
- Add UI smoke-test checklist and manual QA scripts.
- Add crash reporting recommendation with privacy posture.

Avoid:
- Sending clipboard text to logs, crash reports, analytics, or support bundles.
